ListViews and Adapters in Android

Exercise 5: Enable fast scrolling and code a section indexer

This is a code-along lab, which will typically be done in tandem with an online instructor for Xamarin University. Detailed instructions are provided below if you are working through this class on your own.

Duration

15 minutes

Lab goals

The goal here is to improve the user experience when scrolling a ListView through a large data set. You will do this in two ways: enable an enhanced scroll bar with a high rate of motion and provide popup section headers to inform the user of their progress through the data set as they scroll. The finished application is shown below.

Here is a high-level description of the required tasks for this lab:

  1. Use ListView.FastScrollEnabled to turn on fast scrolling.
  2. Implement the ISectionIndexer interface on your custom adapter to provide section headers.

Required assets

The provided Part 05 Resources folder contains resources that you will need in order to complete the lab:

  • The file SectionIndexerBuilder.cs contains a class to perform the tedious parts of implementing a section indexer.
  • The Part5.Start folder contains a starter solution you can use if you did not do the previous exercise or if you would like to begin with a fresh project.
  • The Part5.Completed folder contains a completed version of the lab you can use to check your work.

Please make sure you have this folder before you begin.

Exercise challenge

Use the information in the worksheet to accomplish the tasks outlined above. Alternatively, you can use the step-by-step instructions provided below to work through the exercise.

Steps

Below are the step-by-step instructions to implement the exercise.

Note: you can either continue your work from the prior exercise or use the provided starter solution.

Enable fast scrolling

This section turns on fast scrolling for the instructor ListView. All the functionality here is built in to the ListView class so this requires only a single line of code.

  1. Open MainActivity.cs.
  2. In your OnCreate method, set the FastScrollEnabled property of your instructor ListView to true.
  3. Run the app to test your work. Begin scrolling the ListView as normal, then switch to using the thumb when it appears.
    Note: ListView might not show the thumb even if you have fast scrolling enabled if your data set is not large enough to need it. If the thumb does not appear during your testing, please verify that you have at least three times the amount of data needed to fill the screen.

Implement a section indexer

This part adds a section indexer to your custom adapter. The indexer tells the ListView where the "sections" are in your data. You get to decide what a "section" means; for example, our instructors are sorted by name so the sections will be based on the first letter of their name.

To code a section indexer, your adapter needs to implement the three methods in the ISectionIndexer interface. The code is fairly straightforward but somewhat tedious to implement; therefore, the provided SectionIndexerBuilder class will do most of the work for you.

  1. Add the file SectionIndexerBuilder.cs from the Part 05 Resources folder to the project. Open the file and look at the class definition. There are three helper methods (shown below) that build the data structures you need to easily implement the three ISectionIndexer methods. 
    public static class SectionIndexerBuilder
{
  // builds an array of unique section headers, data must be sorted by name
  public static Java.Lang.Object[] BuildSectionHeaders(List<Instructor> data) ...

  // builds a map to answer: position --> section, data must be sorted by name
  public static Dictionary<int, int> BuildSectionForPositionMap(List<Instructor> instructors) ...

  // builds a map to answer: section --> position, data must be sorted by name
  public static Dictionary<int, int> BuildPositionForSectionMap(List<Instructor> instructors) ...
}
  2. Open InstructorAdapter.cs.
  3. Call each of the three SectionIndexerBuilder methods and store the results in fields inside your adapter class.

    Show Code 

    Java.Lang.Object[] sectionHeaders
  = SectionIndexerBuilder.BuildSectionHeaders(InstructorData.Instructors);

Dictionary<int, int> positionForSectionMap
  = SectionIndexerBuilder.BuildPositionForSectionMap(InstructorData.Instructors);

Dictionary<int, int> sectionForPositionMap
  = SectionIndexerBuilder.BuildSectionForPositionMap(InstructorData.Instructors);

  4. Add ISectionIndexer to the list of your adapter's base types.
  5. Use the results from the SectionIndexerBuilder to implement the three ISectionIndexer methods.

    Show Code 

    public Java.Lang.Object[] GetSections()
{
  return sectionHeaders;
}

public int GetPositionForSection(int section)
{
  return positionForSectionMap[section];
}

public int GetSectionForPosition(int position)
{
  return sectionForPositionMap[position];
}

  6. Run the app to test your work. Start scrolling normally and the thumb should appear. Drag the thumb and the section headers should pop up as you pass into each new section.

Master-Details (optional after-class exercise)

The app currently displays an alert when the user clicks on an item in the list. A more realistic approach would be to navigate the app to a details view of that instructor. The solution for this part includes code to do this. There is a details Activity (InstructorDetailsActivity.cs and InstructorDetails.axml) provided in the Part 05 Resources folder if you would like to add this behavior to your implementation. You will need to modify your ItemClick handler to create an Intent, target the InstructorDetailsActivity, load the Position into the Intent Extras, and launch the Activity. The provided InstructorDetailsActivity code will handle retrieving the position and displaying the instructor details.

Summary

Here you added fast scrolling and a section indexer to your instructor list to make the experience more user-friendly. When there is a small amount of data in the list, the user can scroll normally; however, when the data set gets large, the user has the option of scrolling quickly using the thumb with popup section headers to guide them.
