Managing simple differences between platforms
Often there are slight differences in the design from one platform to another - particularly with spacing or alignment. Xamarin.Forms offers an elegant abstraction that can provide for these differences in the form of the Device.OnPlatform methods. There are two forms:
  • Device.OnPlatform<T> which provides for compile-time values that change based on the running platform. 
    // <T> specification can be inferred as well, but is shown here for clarity.
double spacing = Device.OnPlatform<double>(
          40,  // iOS
          20,  // Android
          20); // Windows Phone
  • Device.OnPlatform(Action,...) which allows for specific logic to be executed based on the running platform. 
    // Can invoke specific logic for each platform
Device.OnPlatform(
          iOS: () => { ... },
          Android: () => { .... },
          WinPhone: () => { ... });

// Or tailor to a platform with defaults for all the others.
Device.OnPlatform(
          iOS: () => { ... },
          Default: () => { .... });

Built-in Device Abstractions
Xamarin.Forms exposes the underlying device through the Device class. It has several methods which support detecting the idiom, platform, invoking a timer, and switching to the UI thread.
See a full list of the Device methods.
Locating Dependencies
Xamarin.Forms includes a built-in dependency manager called DependencyService which locates and then provides access to abstract dependencies provided by the platform specific projects.
To expose a dependency to the system, you create an assembly-level DependencyAttribute which identifies the concrete type.
